# 界面设计规范（美观 · 专业）

## 1. 设计目标

- **一眼读懂**：最新综合分 S、风险色、建议定投、数据截止时间
- **专业可信**：金融仪表盘风格，非「玩具型」高饱和
- **信息密度适中**：卡片 + 图表 + 表格，支持深色模式

---

## 2. 技术选型

| 项 | 选型 |
|----|------|
| 框架 | **Next.js 14** (App Router) |
| 样式 | **Tailwind CSS 3** |
| 组件 | **shadcn/ui**（Button, Card, Tabs, Dialog, Select, Badge, Skeleton） |
| 图表 | **ECharts**（评分历史、回测）+ **lightweight-charts**（K 线，可选） |
| 图标 | Lucide React |
| 字体 | 中文：**Noto Sans SC**；数字：**Tabular nums**（`font-variant-numeric: tabular-nums`） |
| 状态 | TanStack Query（服务端数据缓存） |

---

## 3. 视觉规范

### 3.1 色彩（深色默认）

| 用途 | 色值（示例） |
|------|--------------|
| 背景 | `#0a0e17` / 卡片 `#111827` |
| 边框 | `#1f2937` |
| 主色 | `#3b82f6`（操作、链接） |
| 涨/安全偏低 | `#22c55e` |
| 跌/风险偏高 | `#ef4444` |
| 警告 | `#f59e0b` |
| 正文 | `#e5e7eb` / 次要 `#9ca3af` |

### 3.2 评分 S 色带

| S 区间 | 含义 | Badge |
|--------|------|-------|
| 0–35 | 偏低估/偏安全 | 绿 |
| 35–65 | 中性 | 灰蓝 |
| 65–100 | 偏高估/偏谨慎 | 橙→红 |

### 3.3 排版

- 页面标题：`text-2xl font-semibold tracking-tight`
- 指标数字：`text-3xl font-bold tabular-nums`
- 免责声明：页脚固定小字 `text-xs text-muted-foreground`

## 4. 信息架构（页面）

```text
/                     → 总览 Dashboard
/instruments          → 标的列表（纳指、标普、日经、DAX、基金）
/instruments/[id]     → 标的详情（S、维度雷达、指标表、K 线）
/dca                  → 定投计算器 + 组合配置
/dca/calendar         → 定投日历（高亮定投日、建议额）
/reports              → AI 中文报告列表
/backtest             → 固定 vs 动态回测
/settings             → API、权重只读展示、M/频率、数据源健康
```

## 5. 核心页面线框

### 5.1 总览 Dashboard

```text
┌─────────────────────────────────────────────────────────────┐
│  智能投资评分                                    [设置] [刷新] │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │ 纳指 S72 │ │ 标普 S65 │ │ 日经 S58 │ │ DAX S61  │  ...   │
│  │ ▼ 谨慎   │ │ 中性     │ │ ...      │ │          │        │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘        │
│  数据截至：2026-05-26 07:15 UTC+8  ·  全部 official ✓        │
├─────────────────────────────────────────────────────────────┤
│  今日定投日                                                  │
│  ┌─────────────────────────────────────────────────────┐    │
│  │ 纳指  计划 500 → 建议 320   M=1.5   [查看 AI 解读]    │    │
│  └─────────────────────────────────────────────────────┘    │
├─────────────────────────────────────────────────────────────┤
│  S 历史趋势（折线）              │  宏观快照 VIX / 10Y       │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 标的详情

- 顶部：大号 S + 环形进度或半仪表
- **维度雷达图**：趋势/动量/波动/估值/宏观
- 指标表格：名称、原值、分位、贡献、来源（算/爬）
- K 线 + MA 叠加（lightweight-charts）
- 侧栏：最新 AI 报告摘要 +「全文」Drawer

### 5.3 定投计算器

- 输入：标的（多选）、P、频率、M（1.5 / 2.0 单选）
- 实时预览：`f(S)` 曲线示意 + 当前建议金额滑块
- 现金池余额、下次定投日倒计时

### 5.4 回测

- 参数：标的、年份、P、M、频率
- 双曲线：固定定投 vs 智能定投；表格：年化、最大回撤、投入次数

## 6. 组件清单（shadcn）

- `Card`：所有模块容器
- `Badge`：S 等级、数据 status
- `Tabs`：详情页「评分 | 指标 | K线 | 报告」
- `Select` / `RadioGroup`：频率、M
- `Skeleton`：加载态
- `Alert`：数据 stale、爬取失败
- `Tooltip`：指标释义（PE-TTM、VIX 等）

---

## 7. 交互原则

| 原则 | 实现 |
|------|------|
| 数据透明 | 每数字旁 `(as of 日期)`；hover 显示来源 crawl/compute |
| 保守默认 | 首次进入默认 M=1.5 |
| 防误触 | 修改 M/频率需确认 Dialog |
| 离线友好 | API 失败显示上次缓存 + 黄条警告 |
| 无障碍 | 色带不仅靠颜色，带文字标签 |

---

## 8. AI 报告展示

- 卡片：摘要（2 句）+ `risk_level` Badge
- 折叠：驱动因素 bullets、风险提示
- 页脚：模型名、生成时间、免责声明
- **不**在报告内编辑定投数字（只读展示系统值）

## 10. 与后端契约

- 所有金额、`S` 由 API 返回，前端不算 f(S)（预览可调 `/dca/preview`）
- `GET /health` 驱动顶栏数据状态灯
- 国际化：MVP 仅 **zh-CN**

## 11. 参考气质（非抄袭）

- Bloomberg / TradingView 的**深色信息密度**
- Linear.app 的**简洁边距与圆角**（`rounded-lg`, `border border-border/50`）
- 避免：大面积渐变、emoji 图标、过多动画
